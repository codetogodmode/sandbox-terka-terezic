## Co sem se dnes naučila

### Terminál a navigace 
- `cd` (change directory), `ls` (list), `pwd` (kde jsem), `Ctrl+C` (ukončit program).
- Mezi `cd` a cestou je **mezera** — `cd /Users/...`, ne `cd/Users/...`.
- Terminál **neumí teleportovat** — samotná cesta jako příkaz nefunguje, vrátí *permission denied*.
- Tab autocomplete, šipky pro historii, drag & drop složky z Finderu pro vložení cesty.
- VS Code z terminálu: `open -a "Visual Studio Code" .` (oklika za `code .`, dokud nezprovozním Shell Command).

### Spouštění .NET 
- `dotnet run --project src/PandaNursery` — celý prefix `dotnet` povinný.
- Překlep `donet` → `command not found`.
- Run/Debug ve VS Code spouští **debugger** (`vsdbg`) — funguje, ale výstupy jsou matoucí. V terminálu `dotnet run` je čistší.

### Struktura C# souboru 
```csharp
namespace XYZ;              // škatulka projektu
class Program               // kontejner
{
    static void Main(...)   // entry point
    {
        // tvůj kód
    }
}
```
- C# je **case-sensitive** (`Console` ≠ `console`).
- Každý příkaz končí **středníkem** `;`.
- Chlupaté závorky `{ }` označují blok kódu.
- Komentáře přes `//` kompilátor ignoruje, slouží mně.

### Proměnné a typy 
- Deklarace = rezervace „škatulky" v paměti: `typ jmeno;` (např. `string jmenoPandy;`).
- Bez `=` je škatulka prázdná, do té doby do ní nelze sahat (kompilátor řekne *use of unassigned local variable*).
- Můžu inicializovat při deklaraci: `float prvniCislo = 5;` (Martinova cesta) nebo nechat prázdné a hned naplnit ReadLinem (moje cesta v PandaNursery).
- Typy: **string** (text), **int** (celé číslo), **float** (desetinné), **bool** (true/false).
- Konvence pojmenování = **camelCase** (`jmenoPandy`, `vahaKg`).

### Vstup od uživatele a parsing 
- **`Console.ReadLine()`** = počká na Enter, vrátí **string** (vždycky, i kdyby uživatel napsal číslo).
- **`Console.ReadKey()`** = počká na libovolnou klávesu (mezera, šipka, písmeno).
- Pro číslo: musím **parsovat** — `vahaKg = float.Parse(textovyVstup);`.
- **Pomocná škatulka** (`textovyVstup`, `textoveCislo`) = přestupní stanice mezi ReadLine (vrací string) a Parse (vrací float).
- Bez parse: kompilátor řekne *cannot convert string to float*.

### Locale a manipulace se stringy 
- Mac mám v českém locale → `float.Parse` čeká **čárku** jako desetinný oddělovač.
- `50,5` ✓, `50.5` ✗ → runtime chyba *FormatException*.
- **Stringy jsou immutable** — `Replace`, `Trim`, `ToLower` vrací **nový string**, neměnný původní. Musím přiřadit zpátky: `textovyVstup = textovyVstup.Replace(...)`.
- **`string.Replace("co", "čím")`** — najít/nahradit. Pro odstranění: nahrazení prázdným stringem (`Replace("kg", "")`).
- **`string.Trim()`** — odstraní whitespace **jen na okrajích**, NE písmenka, NE mezery uvnitř (na tom sem se dnes pálila).
- **`string.ToLower()`** — všechno malými písmeny, kvůli case-sensitive porovnání.
- **Method chaining** — `.ToLower().Replace(...).Trim()` zřetězí volání jako pipeline.
- **Limit Replace přístupu** — umí jen přesnou shodu. Pro nevalidní vstup (`kd`, `padesát`, prázdný) padá → robustní řešení až v **S8** (try-catch / TryParse).

### String interpolace 
```csharp
Console.WriteLine($"Tvoje panda: {jmenoPandy}, váha: {vahaKg} kg");
```
- **`$"…"`** zapne interpolaci.
- **`{název}`** se za běhu nahradí hodnotou proměnné.

### Matematika v C# 
- `+` mezi dvěma **floaty** = sečte čísla.
- `+` mezi dvěma **stringy** = **lepí text** (`"12" + "14"` = `"1214"`, ne `26`).
- Proto je důležité parsovat string na float, než s ním počítám.

### Compile-time vs runtime errors 
- **Compile-time chyba** (CS1002, CS0029) = kompilátor odmítne sestavit. Program **nikdy nepoběží**. Příklad: chybí středník.
- **Runtime chyba** (FormatException, NullReferenceException) = program **běžel** a spadl na konkrétním vstupu. Příklad: `float.Parse("abc")`.
- **Pravidlo**: vždycky čti **první chybu**, ne poslední — jedna často spustí lavinu kaskádových chyb.

### Git workflow 
- `git status` → co je modifikované.
- `git add <konkrétní soubor>` → do staging area (NE `git add .`, vleze mi tam i bordel — `.lscache` apod.).
- `git commit -m "Session X: popis CO"` → checkpoint se zprávou (NE „commitni" / „prosím commit" — to Martinovi nic neřekne).
- `git push` → nahraj na GitHub.
- `git stash push -m "popis"` → uloží rozdělanou práci stranou, vrátí pracovní strom do čistého stavu.
- `git stash list` → seznam stashů.
